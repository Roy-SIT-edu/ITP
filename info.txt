Timetable Scheduler - Student-Friendly Code Rundown
===================================================

This file explains the important files changed in the latest improvement pass.
It focuses on the new workflow features: upload preview, demo samples,
constraint editing, schedule comparison, manual timetable moves, explanations,
availability snapshots, and smart issue summaries.

File: ITP Programming App 2/timetable-app/backend/app/services/import_service.py
Purpose:
  Reads Excel files, normalizes column names, validates rows, previews batches,
  and imports valid batches.

Key snippet:
  def preview_input_template_files(self, db, workbooks):
      frames = [(self._read_workbook_frame(BytesIO(workbook_bytes)), source_filename)
                for workbook_bytes, source_filename in workbooks]
      return self._preview_frames(db, frames)

Student explanation:
  A "frame" is a table read from Excel. This function converts uploaded
  workbook bytes into tables, then asks the preview function to validate them.

Key snippet:
  file_summaries = [
      {"filename": source_filename, "rows_read": int(len(frame.index)), "columns": [str(column) for column in frame.columns]}
      for frame, source_filename in frames

Student explanation:
  This creates one summary per uploaded file. It records the filename, number
  of rows, and detected columns so the UI can show a clear import report.

Key snippet:
  session_data, errors = RequirementInputService().validate_upload_rows(db, upload_rows)

Student explanation:
  The preview and import paths both validate rows with the same service. This
  avoids a bug where preview says a file is fine but import later rejects it.

Key snippet:
  "rows_importable": len(session_data) if not errors else 0

Student explanation:
  Uploads are all-or-nothing. If any row has an error, no rows should be
  imported, so importable rows becomes 0 when errors exist.

4. Sample Workbook Files
------------------------

Folder: outputs/sample-input-workbooks/

Files:
  sample_no_constraints_20_rows.xlsx
  sample_soft_constraints_only_20_rows.xlsx
  sample_hard_constraints_20_rows.xlsx

Student explanation:
  These are ready-to-import demo Excel files. Each has 20 rows and uses the
  same Input_Template sheet name expected by the importer.

How they differ:
  - No constraints: flexible rows without preferred/avoid/fixed times.
  - Soft constraints only: flexible rows with preferred and avoided days.
  - Hard constraints: fixed rows with specific valid day/time slots.


5. Verification Commands
------------------------

Backend:
  cd "ITP Programming App 2/timetable-app/backend"

What these prove:
  npm run build checks TypeScript and production bundling.
  python -m pytest checks backend upload, validation, CRUD, solver, database,
  and constraint behavior.
